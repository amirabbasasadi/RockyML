\subsubsection*{template$<$typename T\+\_\+e, int T\+\_\+dim, int T\+\_\+n\+\_\+particles, int T\+\_\+n\+\_\+tribes$>$\newline
class rocky\+::zagros\+::pso\+\_\+tribes\+\_\+mpi$<$ T\+\_\+e, T\+\_\+dim, T\+\_\+n\+\_\+particles, T\+\_\+n\+\_\+tribes $>$}

Particle Swarm M\+PI Implementation Implementation of Tribe\+P\+SO Reference \+: Chen, K., Li, T. and Cao, T., 2006. Tribe-\/\+P\+SO\+: A novel global optimization algorithm and its application in molecular docking. Chemometrics and intelligent laboratory systems, 82(1-\/2), pp.\+248-\/259. 

We devide the particles on each node into {\ttfamily L} Tribes The first optimization layer is parallelized at Thread-\/level using T\+BB The last optimization layer is parallelized at Process-\/level using M\+PI

Each tribe contains Pt particles where Pt = N\+\_\+particles / L Particle p belongs to tribe t if floor(p/\+Pt) = t It\textquotesingle{}s beneficial to keep the particles of a each Tribe close to each other in the memory

By setting L to 1 the algorithm is equivalent to classic P\+SO
